Business Problem

Many organizations organize events such as conferences, concerts, exhibitions, and sports competitions. Managing customer bookings, venue allocation, and payment records manually is inefficient and prone to errors.

This Event Booking Management System stores customer information, event details, venue information, bookings, and payments in a relational database. It also uses Common Table Expressions (CTEs) and SQL Window Functions to analyze booking trends and generate business insights.

---

Database Schema

The database consists of five related tables:

1. Customers
2. Venues
3. Events
4. Bookings
5. Payments

Relationships

- One customer can make many bookings.
- One event can have many bookings.
- One venue can host many events.
- One booking can have one or more payment records.

CTE Implementations

The project demonstrates the following Common Table Expressions:

1. Simple CTE

Displays bookings whose total amount is greater than a specified value.

2. Multiple CTEs

Uses multiple CTEs to identify customers who booked premium events.

3. Recursive CTE

Generates a sequence of booking numbers.

4. CTE with Aggregation

Calculates total revenue generated by each event.

5. CTE with JOIN

Combines customers, bookings, events, and venues into a comprehensive booking report.

Analysis and Findings

Descriptive Analysis

- Multiple customers successfully booked different events.
- Premium events generated higher booking values.
- Several venues hosted different events.

Diagnostic Analysis

- Events with higher ticket prices produced greater revenue.
- Customers purchasing multiple tickets contributed significantly to total sales.
- Popular venues attracted more bookings.

Prescriptive Analysis

- Increase marketing efforts for high-demand events.
- Offer discounts to repeat customers.
- Schedule more events at venues with consistently high attendance.
- Introduce loyalty rewards to improve customer retention.
